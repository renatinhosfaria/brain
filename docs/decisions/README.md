# Architecture Decision Records

## Objetivo

ADRs registram decisões arquiteturalmente significativas do projeto, com o
contexto que motivou a escolha, a decisão tomada e suas consequências.

Uma decisão é arquiteturalmente significativa quando afeta estrutura,
propriedades não funcionais, dependências, interfaces, deployment, segurança,
persistência ou técnicas de construção.

## Quando Criar Um ADR

Crie um ADR quando uma mudança introduzir ou alterar uma decisão difícil de
inferir apenas pelo código, especialmente quando envolver trade-offs duradouros,
contratos públicos, limites de componentes, riscos operacionais ou requisitos de
segurança.

## Formato

Cada ADR deve usar as seções:

- `Status`
- `Contexto`
- `Decisão`
- `Consequências`
